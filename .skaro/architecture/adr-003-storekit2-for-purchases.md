# ADR-003: Использование StoreKit 2 как единственного источника истины для статуса Premium

**Status:** accepted
**Date:** 2026-03-29

## Context
Приложение имеет модель монетизации Free/Premium. Необходимо надёжно определять статус подписки пользователя, предотвращать обход проверки и обеспечивать корректную работу при отсутствии сети. Статус Premium влияет на доступность AI-функций, тем, облачных коннекторов и лимитов загрузки.

## Decision
`Transaction.currentEntitlements` (StoreKit 2) — единственный источник истины для `isPremium`. `PremiumStateValidator` — единственный компонент, определяющий статус. Результат кэшируется в `iCloudSettingsStore` с TTL 24 часа и используется ТОЛЬКО при отсутствии сети. При восстановлении сети — немедленная ревалидация. Синхронизация `isPremium` через CloudKit явно запрещена. Верификация `Transaction.currentEntitlements` выполняется на каждом старте приложения.

## Alternatives
1. **Собственный сервер верификации чеков:** Максимальная защита от мошенничества — отклонён, так как требует серверной инфраструктуры, увеличивает latency, создаёт единую точку отказа и избыточен для приложения без серверного бэкенда.
2. **CloudKit для синхронизации isPremium между устройствами:** Удобная синхронизация — отклонён, так как создаёт уязвимость: злоумышленник может записать `isPremium = true` в CloudKit минуя StoreKit, что полностью обходит защиту монетизации.
3. **UserDefaults с локальной верификацией чека:** Простота реализации — отклонён, так как UserDefaults легко модифицируются на jailbroken устройствах, нет криптографической верификации транзакций.

## Consequences
- Positive: Криптографически верифицированный статус подписки. Автоматическая обработка восстановления покупок. Поддержка семейного доступа. Нативная интеграция с App Store.
- Negative: StoreKit 2 требует iOS 15+. При отсутствии сети необходим кэш (TTL 24ч), что создаёт окно в 24 часа для устаревшего статуса. Тестирование требует Sandbox-окружения.
- Risks: Изменения в StoreKit API в будущих версиях iOS могут потребовать адаптации. Кэш TTL 24ч означает, что отменённая подписка остаётся активной до 24 часов в оффлайн-режиме — приемлемый компромисс.
