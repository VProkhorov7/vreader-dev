# ADR-005: Использование ASWebAuthenticationSession для OAuth вместо WKWebView

**Status:** accepted
**Date:** 2026-03-29

## Context
Приложение интегрируется с тремя OAuth2-провайдерами: Google Drive, Dropbox, OneDrive. Необходимо реализовать безопасный OAuth2 PKCE flow, который не позволяет приложению перехватывать credentials пользователя и соответствует требованиям безопасности App Store.

## Decision
Все OAuth2 авторизации реализуются исключительно через `ASWebAuthenticationSession` с PKCE flow. `OAuthManager` управляет токенами, автоматическим refresh и хранением в `KeychainManager.shared`. Использование `WKWebView` для OAuth явно запрещено на уровне архитектурных инвариантов и проверяется при code review.

## Alternatives
1. **WKWebView с перехватом redirect URL:** Полный контроль над UI авторизации — отклонён, так как нарушает политику App Store (Apple требует `ASWebAuthenticationSession` для OAuth), позволяет приложению потенциально перехватывать credentials, не использует системное хранилище cookies и сессий.
2. **SFSafariViewController:** Системный браузер с общими cookies — отклонён, так как не поддерживает автоматический перехват callback URL без дополнительных схем, менее удобен для PKCE flow и не предоставляет нативного API для получения authorization code.
3. **Собственная реализация OAuth через URLSession:** Максимальный контроль — отклонён, так как требует реализации всего PKCE flow вручную, не использует системные cookies и сессии, значительно увеличивает поверхность для ошибок безопасности.

## Consequences
- Positive: Соответствие требованиям App Store и политике безопасности Apple. Пользователь видит системный диалог авторизации, которому доверяет. Приложение не имеет доступа к credentials. Автоматическое использование системных cookies для SSO.
- Negative: Меньше контроля над UI авторизации — нельзя кастомизировать внешний вид окна браузера. Пользователь перенаправляется из приложения, что может восприниматься как прерывание UX.
- Risks: Изменения в `ASWebAuthenticationSession` API в будущих версиях iOS потребуют адаптации. На iPad в некоторых конфигурациях может потребоваться явное указание `presentationContextProvider`.
