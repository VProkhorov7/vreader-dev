# ADR-002: Использование CloudKit для синхронизации аннотаций и NSUbiquitousKeyValueStore для позиции чтения

**Status:** accepted
**Date:** 2026-03-29

## Context
Пользователи читают книги на нескольких устройствах (iPhone, iPad, Mac Catalyst). Необходима синхронизация позиции чтения, аннотаций и прогресса без собственного серверного бэкенда. Решение должно работать в рамках экосистемы Apple без дополнительных затрат на инфраструктуру.

## Decision
Позиция чтения синхронизируется через `NSUbiquitousKeyValueStore` (быстро, до 1MB, мгновенная доставка). Аннотации и прогресс — через CloudKit (`CKRecord`) с поддержкой push-уведомлений (`CKSubscription`) для мгновенной синхронизации. Конфликты разрешаются через `lamportClock` в каждой `Annotation`. Оффлайн-изменения накапливаются в `PendingChangesQueue` и отправляются при восстановлении сети. Retry: exponential backoff 1s → 2s → 4s → 8s → 16s, максимум 5 попыток.

## Alternatives
1. **Собственный REST-бэкенд (например, на Vapor/Hummingbird):** Полный контроль над логикой синхронизации — отклонён, так как требует инфраструктурных затрат, поддержки сервера, реализации аутентификации и значительно увеличивает time-to-market.
2. **Firebase Firestore:** Зрелая облачная БД с реальным временем — отклонена из-за зависимости от стороннего SDK, передачи пользовательских данных на серверы Google, сложности соответствия GDPR и отсутствия нативной интеграции с Apple-экосистемой.
3. **Только NSUbiquitousKeyValueStore для всего:** Простота реализации — отклонён, так как лимит 1MB недостаточен для хранения аннотаций большой библиотеки, нет поддержки сложных запросов и conflict resolution.

## Consequences
- Positive: Нулевые затраты на инфраструктуру, нативная интеграция с Apple ID, автоматическое шифрование данных, push-уведомления через `CKSubscription` обеспечивают мгновенную синхронизацию.
- Negative: Требует обязательных entitlements (`com.apple.developer.icloud-services`, `com.apple.developer.ubiquity-kvs-identifier`). CloudKit недоступен без Apple ID. Ограниченные возможности запросов по сравнению с реляционными БД.
- Risks: Изменения в политике CloudKit со стороны Apple могут повлиять на работу приложения. Квоты CloudKit могут стать узким местом при большом числе пользователей. Синхронизация `isPremium` через CloudKit явно запрещена — нарушение этого правила создаёт уязвимость для обхода монетизации.
