# ADR-007: Использование Lamport Clock для разрешения конфликтов аннотаций вместо wall clock

**Status:** accepted
**Date:** 2026-03-29

## Context
Пользователи редактируют аннотации на нескольких устройствах. При синхронизации через CloudKit возникают конфликты: одна и та же аннотация может быть изменена на двух устройствах в оффлайн-режиме. Wall clock (системное время) ненадёжен из-за возможного расхождения часов между устройствами (clock skew).

## Decision
Каждая `Annotation` содержит `lamportClock: Int` и `deviceID: String`. `lamportClock` инкрементируется при каждой мутации. Conflict resolution: delta < 5 минут → `.autoMerge` (объединение комментариев с разделителем, выбор большего `lamportClock`); delta > 5 минут → `.userPrompt`; один девайс удалил, другой изменил → `.userPrompt`. Сравнение всегда по `lamportClock`, не по wall clock.

## Alternatives
1. **Last Write Wins по wall clock:** Простота реализации — отклонён для аннотаций, так как clock skew между устройствами может достигать нескольких минут, что приведёт к потере изменений. Применяется только для позиции чтения как менее критичных данных.
2. **Operational Transformation (OT):** Точное слияние текстовых изменений — отклонён, так как избыточно сложен для аннотаций (короткие тексты, редкие конфликты), требует хранения истории операций и значительно увеличивает сложность реализации.
3. **CRDT (Conflict-free Replicated Data Types):** Автоматическое слияние без конфликтов — отклонён, так как требует специализированных структур данных, несовместимых с текущей SwiftData моделью, и избыточен для сценария использования (редкие конфликты аннотаций).

## Consequences
- Positive: Защита от clock skew между устройствами. Детерминированное разрешение конфликтов. Пользователь вовлекается в разрешение только при значимых расхождениях (delta > 5 минут).
- Negative: `lamportClock` необходимо инкрементировать при каждой мутации — пропуск инкремента нарушает логику разрешения конфликтов. Автоматическое слияние комментариев с разделителем может выглядеть неестественно для пользователя.
- Risks: Если `lamportClock` не инкрементируется корректно (например, при bulk import), conflict resolution может работать некорректно. Порог 5 минут для `.autoMerge` vs `.userPrompt` выбран эмпирически и может потребовать корректировки по результатам пользовательского тестирования.
