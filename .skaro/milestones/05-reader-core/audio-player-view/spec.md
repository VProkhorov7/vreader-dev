# Specification: audio-player-view

## Context
Аудиоплеер для MP3, M4A, M4B, AAC. Управление воспроизведением, главы, скорость, таймер сна. Интеграция с Control Center и Now Playing. Существующий `AudioPlayerView.swift` требует ревизии.

## User Scenarios
1. **Пользователь слушает аудиокнигу:** Видит обложку, название, прогресс, кнопки управления.
2. **Пользователь сворачивает приложение:** Воспроизведение продолжается, управление через Control Center.
3. **Пользователь переходит к главе 5:** Тапает на главу в списке → воспроизведение начинается с начала главы.

## Functional Requirements
- FR-01: `AudioPlayerView` — SwiftUI View, принимает `book: Book`, `handler: AudioHandler`.
- FR-02: Большая обложка книги в центре.
- FR-03: Кнопки: play/pause, prev chapter, next chapter, rewind 15s, forward 30s.
- FR-04: Прогресс slider с текущим временем и общей длительностью.
- FR-05: Список глав (sheet): название главы, длительность, текущая глава выделена.
- FR-06: Скорость воспроизведения: 0.5x, 0.75x, 1x, 1.25x, 1.5x, 2x.
- FR-07: Таймер сна: 15, 30, 45, 60 минут, конец главы.
- FR-08: `PlayerState` обновляется при изменении воспроизведения.
- FR-09: Now Playing Info обновляется в реальном времени.
- FR-10: Сохранение позиции при закрытии через `ReadingStateManager`.
- FR-11: Все строки через `L10n.*`.

## Non-Functional Requirements
- NFR-01: Фоновое воспроизведение без прерываний.
- NFR-02: Переход между главами < 500ms.

## Boundaries (что НЕ входит)
- Не реализовывать TTS в AudioPlayerView — это TTSService.
- Не реализовывать Immersion Reading (синхронизация текста) — milestone 10.

## Acceptance Criteria
- [ ] Воспроизведение M4B с главами работает.
- [ ] Фоновое воспроизведение работает.
- [ ] Control Center управляет воспроизведением.
- [ ] Скорость воспроизведения изменяется.
- [ ] Таймер сна работает.
- [ ] Существующий AudioPlayerView.swift обновлён.

## Open Questions
- Нужна ли визуализация звуковой волны (waveform)?
- Как обрабатывать прерывания (звонок, другое аудио) — пауза или продолжение?
