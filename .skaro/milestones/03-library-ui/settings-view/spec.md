# Specification: settings-view

## Context
SettingsView — центр управления приложением. Настройки темы, шрифта, языка сохраняются в iCloudSettingsStore и синхронизируются между устройствами. Облачные коннекторы управляются здесь. DiagnosticsView доступна в Debug режиме полностью, в Release — только экспорт.

## User Scenarios
1. **Пользователь меняет тему:** Выбирает из доступных тем, Premium темы заблокированы с замком.
2. **Пользователь подключает Google Drive:** Переходит в CloudConnectorView из Settings.
3. **Пользователь экспортирует логи:** Нажимает "Экспорт диагностики" → share sheet.

## Functional Requirements
- FR-01: List с секциями: Внешний вид, Чтение, Облако, Premium, Диагностика, О приложении
- FR-02: Секция Внешний вид: выбор темы (4 варианта с превью), размер шрифта (slider), межстрочный интервал
- FR-03: Premium темы показываются с иконкой замка, тап → PremiumPaywallView (заглушка)
- FR-04: Секция Чтение: язык интерфейса, автопрокрутка (toggle, milestone 10)
- FR-05: Секция Облако: список подключённых провайдеров со статусом, кнопка "Добавить"
- FR-06: Секция Premium: статус подписки, кнопка "Управление подпиской"
- FR-07: Секция Диагностика: в Debug — полный DiagnosticsView, в Release — кнопка "Экспортировать логи"
- FR-08: Секция О приложении: версия, build, ссылки
- FR-09: Изменения темы применяются немедленно через iCloudSettingsStore
- FR-10: Все строки через L10n.Settings.*

## Non-Functional Requirements
- NFR-01: Изменение темы применяется < 100ms
- NFR-02: Настройки синхронизируются через iCloudSettingsStore

## Boundaries (что НЕ входит)
- Не реализовывать CloudConnectorView полностью (заглушка)
- Не реализовывать PremiumPaywallView (заглушка)
- Не реализовывать полный DiagnosticsView (только экспорт)

## Acceptance Criteria
- [ ] Все секции отображаются
- [ ] Смена темы применяется немедленно
- [ ] Premium темы заблокированы с замком
- [ ] Экспорт логов через share sheet работает
- [ ] Настройки сохраняются в iCloudSettingsStore
- [ ] Все строки через L10n
