# Specification: library-view

## Context
LibraryView — главный экран приложения. Отображает все книги пользователя в виде сетки. Поддерживает поиск, фильтрацию и сортировку. Три вкладки: все книги, избранное, коллекции. Использует @Query для загрузки из SwiftData с P95 < 300ms для 1000 книг.

## User Scenarios
1. **Пользователь открывает приложение:** Видит сетку книг с обложками, отсортированных по дате добавления.
2. **Поиск "Толстой":** Список фильтруется по title и author в реальном времени.
3. **Фильтр по жанру "Fantasy":** Показываются только книги жанра Fantasy.
4. **Импорт новой книги:** Книга появляется в сетке без перезагрузки.

## Functional Requirements
- FR-01: LazyVGrid с адаптивными колонками (минимум 120pt ширина карточки)
- FR-02: @Query для загрузки Book из SwiftData с сортировкой
- FR-03: Поиск по title, author, tags через Predicate
- FR-04: Фильтры: по жанру, по формату (BookFormat), по источнику (ContentSource), по contentState
- FR-05: Сортировка: по дате добавления, по дате открытия, по названию, по автору, по прогрессу
- FR-06: Три сегмента: "Все" (все книги), "Избранное" (isFinished или специальный тег), "Коллекции" (список Collection)
- FR-07: Кнопка импорта (fileImporter) в toolbar
- FR-08: Long press на карточке → контекстное меню: Открыть, Редактировать метаданные, Удалить, Поделиться, Обновить метаданные
- FR-09: Pull-to-refresh для синхронизации с облаком
- FR-10: Пустое состояние: иллюстрация + текст + кнопка импорта
- FR-11: Пагинация при > 500 книг (fetchLimit + fetchOffset)
- FR-12: LibraryState используется для хранения searchQuery и activeFilters
- FR-13: Поддержка iPad: 4-5 колонок в landscape
- FR-14: Все строки через L10n.Library.*

## Non-Functional Requirements
- NFR-01: P95 < 300ms для отображения 1000 книг
- NFR-02: Плавный scroll без jank (LazyVGrid + AsyncImage)
- NFR-03: Поиск debounce 300ms

## Boundaries (что НЕ входит)
- Не реализовывать OPDS каталоги (это CatalogsView)
- Не реализовывать облачный браузер (это CloudFileBrowserView)
- Не реализовывать ридер (это ReaderView)

## Acceptance Criteria
- [ ] LazyVGrid с адаптивными колонками работает
- [ ] Поиск фильтрует в реальном времени
- [ ] Все фильтры и сортировки работают
- [ ] Три сегмента переключаются
- [ ] Контекстное меню на long press работает
- [ ] Пустое состояние отображается
- [ ] Пагинация при > 500 книг
- [ ] P95 < 300ms для 1000 книг
