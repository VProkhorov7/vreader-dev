# Specification: cloud-file-browser

## Context
UI для просмотра файлов в облачных провайдерах и управления подключениями. Пользователь выбирает файлы для загрузки в библиотеку. Существующие `CloudFileBrowserView.swift`, `CloudStorageView.swift` требуют ревизии.

## User Scenarios
1. **Пользователь открывает Dropbox:** Видит список файлов и папок, может навигировать.
2. **Пользователь выбирает книгу:** Файл добавляется в очередь загрузки через DownloadManager.
3. **Пользователь управляет подключениями:** CloudStorageView показывает все провайдеры, статус, кнопки подключить/отключить.

## Functional Requirements
- FR-01: `CloudFileBrowserView` — SwiftUI View, принимает `provider: any CloudProviderProtocol`.
- FR-02: Список файлов и папок с иконками типов.
- FR-03: Навигация по папкам (breadcrumb).
- FR-04: Тап на файл книги → `DownloadManager.download(book:)` → добавление в библиотеку.
- FR-05: Тап на папку → навигация внутрь.
- FR-06: Pull-to-refresh для обновления листинга.
- FR-07: Индикатор загрузки при PROPFIND/listFiles.
- FR-08: `CloudStorageView` — список всех провайдеров: иконка, название, статус, кнопка подключить/отключить.
- FR-09: Кнопка 'Подключить' → OAuth flow или WebDAV форма ввода credentials.
- FR-10: Статус провайдера из `CloudProviderHealthMonitor`.
- FR-11: Все строки через `L10n.*`.

## Non-Functional Requirements
- NFR-01: Листинг 100 файлов отображается < 2s.

## Boundaries (что НЕ входит)
- Не реализовывать загрузку файлов на провайдер из приложения.
- Не реализовывать поиск по файлам провайдера.

## Acceptance Criteria
- [ ] Файлы провайдера отображаются корректно.
- [ ] Навигация по папкам работает.
- [ ] Выбор файла запускает загрузку.
- [ ] CloudStorageView показывает все провайдеры и их статус.
- [ ] Существующие файлы обновлены.

## Open Questions

- Как отображать файлы неподдерживаемых форматов — скрывать или показывать серыми?
