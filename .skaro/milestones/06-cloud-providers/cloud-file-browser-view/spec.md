# Specification: cloud-file-browser-view

## Context
CloudFileBrowserView позволяет пользователю просматривать файлы в подключённых облачных хранилищах и импортировать книги. Показывает статус провайдера (active/degraded) и contentState каждой книги.

## User Scenarios
1. **Пользователь открывает Google Drive:** Видит список папок и файлов, может навигировать по папкам.
2. **Пользователь нажимает на книгу:** Книга добавляется в библиотеку с contentState = .cloudOnly, начинается фоновая загрузка превью.
3. **Провайдер деградирован:** Показывается banner с предупреждением.

## Functional Requirements
- FR-01: NavigationStack для навигации по папкам
- FR-02: Список файлов и папок с иконками типов
- FR-03: Фильтрация: показывать только поддерживаемые форматы книг + папки
- FR-04: Тап на книгу → добавить в библиотеку (Book с contentState = .cloudOnly) + schedulePreview
- FR-05: Тап на папку → навигация внутрь
- FR-06: Pull-to-refresh для обновления списка
- FR-07: Статус провайдера в navigation bar (зелёный/красный индикатор)
- FR-08: Banner при degraded статусе провайдера
- FR-09: Поиск по имени файла
- FR-10: Кнопка "Импортировать всю папку" → пакетный импорт
- FR-11: Все строки через L10n.Cloud.*

## Non-Functional Requirements
- NFR-01: Загрузка списка файлов < 3 секунды
- NFR-02: Пагинация для больших директорий

## Boundaries (что НЕ входит)
- Не реализовывать загрузку файлов на облако
- Не реализовывать редактирование файлов в облаке

## Acceptance Criteria
- [ ] Навигация по папкам работает
- [ ] Импорт книги добавляет в библиотеку
- [ ] Статус провайдера отображается
- [ ] Degraded banner показывается
- [ ] Поиск работает
- [ ] Все строки через L10n
